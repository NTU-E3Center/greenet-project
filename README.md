## 1. 系統概覽

### 1.1 平台簡介

綠電轉供最佳化平台協助天能售電業者完成從數據模擬到智慧決策的完整綠電轉供分析流程。平台包含三大模組：

| 模組 | 功能 | 適用場景 |
|------|------|----------|
| **M1：用電數據模擬** | 將月度電費單轉換為 15 分鐘精度的用電曲線 | 新客戶尚無精細用電數據時 |
| **M2：轉供結算** | 依台電規章進行兩階段媒合結算 | 每月結算作業、試算不同情境 |
| **M3：轉供最佳化** | 找出最大化毛利的最佳配對方案 | 年度規劃、新案場/新客戶進場 |

### 1.2 導航方式

- **點擊左上角 Logo**（⚡ 綠能轉供平台）可隨時返回首頁
- **左側選單** 可切換三個模組（M1 / M2 / M3）
- 首頁的三張卡片也可以直接點擊進入對應模組

## 2. 模組一：用電數據模擬

> **目的**：為缺乏 15 分鐘精度歷史用電數據的客戶，模擬其用電曲線。

### Step 1：輸入用電數據

1. 點擊左側 **M1：用電數據模擬**
2. 在 Step 1 的表格中，填入客戶過去 12 個月的電費金額（單位：元）：

| 欄位 | 說明 |
|------|------|
| 🔴 尖峰（元） | 尖峰時段電費金額 |
| 🟠 半尖峰（元） | 半尖峰時段電費金額 |
| 🟢 離峰（元） | 離峰時段電費金額 |
| 小計 | 各時段自動加總 |

3. 表格底部顯示 **合計** 列，自動彙整 12 個月各時段總金額
4. 點擊 **下一步** 進入 Step 2

### Step 2：選擇用戶類型

1. 從四種用戶類型卡片中 **點選一張** 最符合客戶用電特性的類型：

| 類型卡片 | 用電特徵 | 典型案例 |
|----------|----------|----------|
| 🏭 24 小時型 | 全天候穩定用電 | 工廠、數據中心 |
| 🏢 辦公室型 | 平日 08:00–17:00 集中用電 | 企業總部、辦公大樓 |
| 🏪 門市型 | 一週七天，白天為主 | 便利商店、銀行分行 |
| 🏬 百貨型 | 平日晚間及假日用電較高 | 購物中心、百貨公司 |

2. 點擊 **執行模擬** 後，系統將生成全年 35,040 筆 15 分鐘時序資料
3. 模擬完成後可：
   - **📥 匯出 CSV**：將時序資料下載存入資料庫

> **💡 提示**：圖表支援「全年總覽」與「單日放大」兩種檢視模式，拖動時間軸可放大特定時段。

## 3. 模組二：轉供結算

> **目的**：依照台電《電能轉供與併網型直供營運規章》第 13 條，進行兩階段媒合結算。

### Step 1：選擇公司

1. 點擊左側 **M2：轉供結算**
2. 左邊勾選參與本次結算的 **發電端企業**
3. 右邊勾選參與本次結算的 **用電端企業**
4. 如需新增不在列表中的企業，點擊 **➕ 手動新增** 按鈕填寫資訊

**企業名稱顯示格式**：

- 發電端：電號+企業名稱+容量kW（收購價 X.X）
`☀️ GEN-001 天能太陽能一廠 100kW（收購價 5.1）`
- 用電端：電號+企業名稱+售電價（售電價 X.X）
`🏭 USR-001 全聯實業（售電價 5.5）`

> **📦 DB 標籤**：表示資料來源為資料庫自動帶入  
> **✋ 手動 標籤**：表示使用者手動新增的企業

### Step 2：時間範圍

1. 選擇計算的 **起始月份** 和 **結束月份**
2. 系統自動從資料庫載入所選時段的 15 分鐘發電/用電數據
3. 點擊 **下一步**

### Step 3：契約設定

設定各企業的轉供相關參數：

| 設定項目 | 說明 | 預設值 |
|----------|------|--------|
| 裝置容量 (kW) | 發電端設備容量 | 由 DB 帶入，可修改 |
| 收購價 (元/度) | 天能收購每度電的價格 | 由 DB 帶入，可修改 |
| 轉供比例 (%) | 發電量中用於轉供的比例 | 100% |
| 售電價 (元/度) | 天能售出每度電的價格 | 由 DB 帶入，可修改 |
| 月上限 (kWh) | 每月約定轉供上限度數 | 使用者設定 |
| 年上限 (kWh) | 每年約定轉供上限度數 | 使用者設定 |

### Step 4：確認輸入

- 系統以摘要形式顯示所有設定
- 此步驟的數值 **仍可直接修改**
- 確認無誤後點擊 **執行結算**

### Step 5：結算結果

- 上方顯示摘要 KPI（總轉供量、總毛利等）
- 可切換 **總覽** 及各 **月份分頁** 查看詳細結算明細
- 結果包含第一階段（15 分鐘即時媒合）和第二階段（同電價時段餘電再媒合）

## 4. 模組三：轉供最佳化

> **目的**：在數萬種發電-用電組合中，利用最佳化演算法找出最大化毛利的最佳配對方案。

### Step 1：選擇範圍

1. 點擊左側 **M3：轉供最佳化**

2. **不考慮已簽轉供契約發電端（選用）**：  
   頁面頂部有一個琥珀色開關 **「不考慮已簽轉供契約發電端」**。勾選後，已簽台電轉供契約之案場（標記為 `已簽轉供契約`）將自動變為灰色、無法選取，不參與最佳化計算。取消勾選後恢復正常。

3. 左側勾選要參與優化的 **發電端案場**
4. 右側勾選要參與優化的 **用電端企業**
5. 如需新增，可點擊 **➕ 手動新增** 按鈕

> **標記說明**：  
> - `已簽轉供契約`（紅色標籤）：該發電端已簽署台電轉供契約  

### Step 2：設定約束

#### ⚙️ 全域參數

位於頁面頂部的全域參數卡片，對整體最佳化有約束效果：

| 參數 | 說明 |
|------|------|
| **最大總契約數** | 演算法產出的契約總數不會超過此上限 |
| **發電契約可拆分數** | 每個發電端可被拆分配對至的最大用電端數量 |

#### 發電端約束

| 欄位 | 說明 |
|------|------|
| 電號 | 發電端識別碼（DB自動帶入） |
| 保收量 (kWh) | 年度保證收購量目標 |

#### 用電端約束

| 欄位 | 說明 | 操作方式 |
|------|------|----------|
| 電號 | 用電端識別碼 | DB自動帶入 |
| 保供量 (kWh) | 年度保證供電量目標 | 直接在欄位中修改數值 |
| **電號綁定模式** | 電號分組契約方式 | **下拉選單** |

#### 電號綁定模式說明

| 模式 | 說明 | 適用場景 |
|------|------|----------|
| **獨立** | 每個電號各自獨立分配 | 不同性質的用電端 |
| **同企業** | 同一企業的所有電號必須綁定在同一組配對 | 同一公司的多個分支機構 |
| **允許跨企業** | 不同企業的電號也可綁成一組 | 關係企業或集團公司 |

### Step 3：確認輸入 + 執行最佳化

1. 系統以摘要表格顯示所有設定：
   - **全域參數**（最大總契約數、發電契約可拆分數）
   - **發電端案場** 確認表（電號、名稱、容量、收購價、保收量、資料來源）
   - **用電端企業** 確認表（電號、名稱、售電價、保供量、電號綁定模式、資料來源）
2. **此頁面的數值仍可直接修改**
3. 確認無誤後，點擊底部的 **🚀 開始最佳化** 按鈕
4. 系統將顯示進度條及執行狀態
5. 完成後出現 **查看結果 →** 按鈕

### Step 4：查看結果

最佳化結果頁面包含：

**上方 KPI 摘要卡片**：

| KPI | 說明 |
|-----|------|
| 總毛利 | 所有契約的毛利加總 |
| 轉供量 | 總配對成功的電力度數 |
| 餘電量 | 發電端未被配對的剩餘電力 |
| 缺口量 | 用電端未獲分配的電力缺口 |
| 契約數 | 產生的契約組合數量 |

**下方契約分配明細表**：

| 欄位 | 說明 |
|------|------|
| 契約 | 契約編號（如 OPT-001）及配對型態（如 1對3、2對1） |
| 發電端 | 配對的發電案場名稱與容量 |
| 轉供比例 | 該發電端在此契約中分配的比例（同一發電端跨契約加總 = 100%） |
| 用電端 | 配對的用電企業名稱 |
| 月上限 (kWh) | 該配對的每月轉供上限 |
| 年上限 (kWh) | 該配對的每年轉供上限 |
| 預估轉供量 | 預估的電力轉供度數 |
| 預估毛利 | 該配對的預估毛利金額 |



完成後可點擊：
- **← 重新設定**：返回 Step 1 重新調整參數
- **📥 匯出結果**：將最佳化配對結果匯出
